настраиваем имя пользователя и почту, проверяем, что настройки ок
[serge] [gentoo]:~ $ git config --global user.name "Serge Zakhry"
[serge] [gentoo]:~ $ git config --global user.email "legrente@gmail.com"
[serge] [gentoo]:~ $ git config --list

далее нужно перейти в папку, которую хотим добавить в реп и проверить, есть ли гит в ней и какой их статус
$ git status

далее инициализируем репу 
$ git init

добавляем все из папки в репу (добавление в индекс)
$ git add .

и коммитим (сохраняем изменения в репе. сохраняются только те файлы, которые были добавлены по git add)
$ git commit -m 'initial'

смотрим, что закоммитилось
$ git show

меняем файлик, добавляем в реп, коммитим и смотрим
$ git add filename
$ git commit -m 'git.txt update'
$ git log - посмотреть список коммитов
$ git show “хеш коммита” - посмотреть инфу по конкретному коммиту


ИГНОРИРОВАНИЕ ФАЙЛОВ

- логи
- пользовательские файлы
- служебные файлы среды разработки
- внешние библиотеки
- файлы локальных конфигураций
- файлы ОС
- слишком большие файлы

В директории создается файл .gitignore и туда с новой стройки помещаются все имена фалойлов и директорий, которые нужно игнорить

можно использовать маски. например *.log - будут игнориться все логи
директорию тоже можно исключить - на конце слеш / поставить
также можно исключать в гитигнор - надо поставить восклицательный знак к пути из папки, где находится гит репозиторий
